# 井轨迹优化结果可视化使用说明

## 概述

提供了两个可视化脚本：
1. `visualize_optimization_result.py` - 直接使用参数值可视化
2. `visualize_from_result.py` - 从结果文件读取参数并可视化

---

## 方法1: 直接使用参数值

### 基本用法

```bash
python visualize_optimization_result.py \
  --optimal-params "950,70,90,200,90,1200,400,2450" \
  --target-e 502.64 \
  --target-n 790.71 \
  --target-d 2636.06 \
  --wellhead 254.0,2030.0,0.0 \
  --excel-files 米41-37YH3-simple.xlsx 米41-37YH5-simple.xlsx \
  --wellhead-positions "208,2015,0" "210,2007,0" \
  --save-path well_trajectory_result.png
```

### 参数说明

- `--optimal-params`: 最优参数，格式：`D_kop,alpha1,alpha2,phi1,phi2,R1,R2,D_turn_kop`
- `--target-e/n/d`: 目标点坐标
- `--wellhead`: 井口坐标，格式：`e,n,d`
- `--excel-files`: 已有井Excel文件列表
- `--wellhead-positions`: 已有井井口位置列表
- `--save-path`: 保存路径（可选）
- `--no-show`: 不显示图表（仅保存）
- `--no-param-info`: 不显示参数信息图

## 方法2: 从结果文件读取

### 支持的文件格式

#### JSON格式
```json
{
  "best_solution": [950, 70, 90, 200, 90, 1200, 400, 2450],
  "best_fitness": 1234.56,
  "trajectory_info": {...}
}
```

#### 文本格式（每行一个参数值）
```
950
70
90
200
90
1200
400
2450
```

#### CSV格式（包含参数列）
```csv
D_kop,alpha1,alpha2,phi1,phi2,R1,R2,D_turn_kop
950,70,90,200,90,1200,400,2450
```

### 使用示例

```bash

  --target-n 790.71 \

  --wellhead-positions "208,2015,0" "210,2007,0" \
```

## 快速使用示例

### 从B2OPT推理结果可视化

如果B2OPT推理输出了结果，可以保存为JSON：

```python
# 在推理脚本中添加
import json

result = optimize_with_b2opt(...)
with open('optimization_result.json', 'w') as f:
    json.dump({
        'best_solution': result['best_solution'].tolist(),
        'best_fitness': result['best_fitness'],
        'trajectory_info': result['trajectory_info'],
    }, f, indent=2)
```

然后可视化：

```bash
python well_trajectory_objective/visualize_from_result.py \
  --result-file optimization_result.json \
  --target-e 502.64 \
  --target-n 790.71 \
  --target-d 2636.06 \
  --wellhead 254.0,2030.0,0.0 \
  --excel-files 米41-37YH3-simple.xlsx 米41-37YH5-simple.xlsx \
  --wellhead-positions "208,2015,0" "210,2007,0" \
  --save-path well_trajectory_optimization.png
```

## Python代码中使用

```python
from well_trajectory_objective import WellTrajectoryConfig, create_multiple_well_obstacles
from well_trajectory_objective.visualize_optimization_result import visualize_optimization_result

# 最优参数
optimal_params = [950, 70, 90, 200, 90, 1200, 400, 2450]

# 创建配置
config = WellTrajectoryConfig(
    E_target=248.64,  # 相对目标点
    N_target=-1239.29,
    D_target=2636.06,
    E_wellhead=254.0,
    N_wellhead=2030.0,
    D_wellhead=0.0,
)

# 创建已有井障碍物
well_obstacles = create_multiple_well_obstacles(
    ["米41-37YH3-simple.xlsx", "米41-37YH5-simple.xlsx"],
    wellhead_positions=[(208, 2015, 0), (210, 2007, 0)],
    safety_radius=10.0,
    segment_length=150.0,
)

# 可视化
visualize_optimization_result(
    optimal_params=optimal_params,
    config=config,
    well_obstacles=well_obstacles,
    wellhead_position=(254.0, 2030.0, 0.0),
    save_path="result.png",
    show=True,
)
```

## 可视化特性

### 颜色编码

- **蓝色**: 垂直段
- **橙色**: 造斜段
- **绿色**: 第一切线段
- **红色**: 转向段
- **紫色**: 第二切线段
- **红色圆点**: 井口
- **绿色方块**: 目标点
- **橙色线**: 已有井轨迹

### 参数验证

- ✅ 自动检查参数是否在配置的取值范围内
- ⚠️ 超出范围的参数用红色标记
- ✓ 在范围内的参数用蓝色标记

## 注意事项

1. **坐标系统**: 
   - 井口和目标点使用绝对坐标
   - 配置中使用相对坐标（目标点 - 井口）

2. **已有井文件**: 
   - 确保Excel文件路径正确
   - 井口位置数量需与Excel文件数量匹配

3. **参数格式**: 
   - 参数顺序：`[D_kop, alpha1, alpha2, phi1, phi2, R1, R2, D_turn_kop]`
   - 角度单位为度（不是弧度）

---

## 故障排除

### 问题1: 无法计算轨迹

**原因**: 参数可能无效或超出范围

**解决**: 检查参数是否在配置的取值范围内

### 问题2: 已有井不显示

**原因**: Excel文件路径错误或格式不正确

**解决**: 检查文件路径和格式

### 问题3: 图像显示不完整

**原因**: 图形窗口太小

**解决**: 调整 `figsize` 参数或使用 `--save-path` 保存高分辨率图像
